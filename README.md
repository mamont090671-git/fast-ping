# fast-ping

Многопоточный ping-сканер на C для быстрой проверки доступности нескольких IP-адресов или доменных имен.

## Особенности

- **Многопоточность** — проверка нескольких адресов параллельно
- **Цветовой вывод** — зелёный [ ONLINE ], красный [ OFFLINE ]
- **Двуязычный вывод** — русский и английский
- **Обработка прерывания** — корректное завершение при Ctrl+C
- **Использует ping CLI** — через `popen()`, не требует root-прав
- **Обработка ошибок** — валидация аргументов, статусов выполнения
- **Итоговая сводка** — Total/Success/Failed/Loss %

## Компиляция

```bash
gcc -o fast-ping fast-ping.c -pthread
```

## Использование

```bash
./fast-ping [OPTIONS] -c <packets> <ip1> <ip2> ...
```

### Опции

- `-c <count>` — количество ICMP-пакетов для каждого адреса (ОБЯЗАТЕЛЬНО)
- `-l <lang>` — язык вывода: `0` = English (по умолчанию), `1` = Russian

### Примеры

```bash
# Проверить 3 пакета до google.com и yandex.ru на английском (по умолчанию)
./fast-ping -c 3 google.com yandex.ru

# Проверить 3 пакета на русском языке
./fast-ping -l 1 -c 3 google.com yandex.ru

# Проверить 5 локальных адресов
./fast-ping -l 0 -c 5 192.168.1.1 192.168.1.2 192.168.1.3

# Проверить с смешанными доменами и IP
./fast-ping -c 2 google.com 192.168.1.1
```

## Вывод (английский)

```bash
$ ./fast-ping -l 0 -c 2 google.com
STATUS      IP ADDRESS      | TIME (min/avg/max/mdev)
------------------------------------------------------------
[ ONLINE  ] google.com      | RTT (ms): 78.923/89.270/99.617/10.347 ms
[ OFFLINE ] 192.168.1.100   | Unavailable
------------------------------------------------------------
```

## Вывод (русский)

```bash
$ ./fast-ping -l 1 -c 2 google.com
СТАТУС    IP АДРЕС    | ВРЕМЯ (min/avg/max/mdev)
------------------------------------------------------------
[ ONLINE  ] google.com      | RTT (ms): 78.923/89.270/99.617/10.347 ms
[ OFFLINE ] 192.168.1.100   | Недоступен
------------------------------------------------------------
```

## Аргументы

- `-c <count>` — количество ICMP-пакетов для каждого адреса
- `<ip>` — IPv4-адрес или доменное имя
- `-l <lang>` — язык: `0` = English, `1` = Russian

## Ограничения

- Работает только на Linux (использует `LC_ALL=C ping`)
- Требует права на выполнение утилиты `ping`
- Для IPv6 используйте `ping6` (нужна модификация кода)

## Обработка ошибок

- **Ctrl+C** — корректное завершение всех потоков
- **Неверный формат аргументов** — вывод помощи и выход с кодом 1
- **Ошибки выполнения** — сообщение в stdout с цветом ERROR

## Архитектура

- Каждый IP проверяется в отдельном потоке
- Использует `popen()` для запуска системной утилиты `ping`
- Парсит вывод `ping` для извлечения RTT statistics
- `volatile int cancelled` для сигналов SIGINT/SIGTERM
- Две версии вывода: английский и русский, выбирается при компиляции

## Зависимости

- GCC
- POSIX threads (`-pthread`)
- Linux system `ping` utility
